用hexo+GitHub搭建博客

发表于2021-02-08|更新于2022-11-14
|阅读量:

1.安装nodejs

进入nodejs.org/en/download/，如果太慢了就进入http://nodejs.cn/download/， 选择相应的版本

安装完成后在命令行检查

由于安装包的时候默认下载到C:\Users\用户名\AppData\Roaming\npm，所以要修改全局模块安装目录：在node.js安装目录下新建两个文件夹 node_global和node_cache

在命令行输入：npm config set prefix “创建的node_global文件夹所在路径” ， npm config set cache “创建的node_cache文件夹所在路径”

然后配置环境变量：右键此电脑——高级系统设置——环境变量——系统变量——新建

NODE_PATH，值为E: \node\node_global\node_modules

环境变量——用户变量——编辑Path，将相应npm的路径（“C:\Users\用户名\AppData\Roaming\npm”）改为：“E: \node\node_global”

2.下载hexo

利用npm下载镜像，由于国内比较慢，先安装cnpm淘宝源:

命令行输入npm install -g cnpm –registry=https://registry.npm.taobao.org

但是输入cnpm时报错，进入global文件夹再试一次又有了

说明没有配置环境变量，编辑系统变量中的path，加上cnpm的路径E:/node/node_global

用cnpm安装hexo：命令行输入cnpm install –g hexo-cli

3.搭建本地博客

新建一个文件夹blog，在命令行进入这个文件夹（很重要，凡是写博客都一定要先进入文件夹），然后输入hexo init初始化博客（前提是先把git装好），如果太慢了就直接在blog文件夹下右键git Bash然后再输入这个命令

启动hexo：hexo s，在浏览器中输入显示的端口

4.部署到GitHub

在部署之前先敲代码

git config –global user.email “xxx” 邮箱
git config –global user.name “xxx” 仓库名

新建仓库：new repository 命名为：用户名.github.io（必须是这个名字），新建仓库以后复制新仓库的网址，就是clone按钮下面的网址

进入命令行：在blog录下 安装git部署插件cnpm install –save hexo-deployer-git

在blog文件夹下打开_config.yml文件，在最底部type:git，提行repo:复制的网址，

提行branch:master（不写也默认是这个）

如果不想输入密码，那就配置git的ssh加密通信，在git Bash（一定要是主目录下）使用以下命令生成 SSH Key：$ ssh-keygen -t rsa -C “youremail@example.com“，然后一路回车

生成的密钥会默认放在C:\Users\CC，进入文件夹打开里面的.pub文件并复制

回到 github 上，进入 Account => Settings：点击进入SSH那一栏，title随便写，把复制的密钥粘贴上去：

为了验证是否成功，输入以下命令：ssh -T git@github.com

用hexo clean清除缓存，hexo g生成网页，hexo d进行部署，此时更奇葩的问题出现了，部署的时候进行到拷贝的那个步骤时直接终止了：

解决办法如下：先卸载原来的部署插件hexo-deployer-git，然后直接在master分支安装： npm un hexo-deployer-git

​ npm i hexojs/hexo-deployer-git

如果hexo d部署之后远程博客仍然没有更新，那就先删除.deploy_git的文件夹，这个是前面的主机生成的网站项目内容，与当前的不兼容。

完了以后博客的地址就是仓库后面的斜杠：corangedas.github.io

5.给博客换主题

lilya：

先找到想要的主题的连接。如https://github.com/litten/hexo-theme-yilia，然后再博客目录下用git来拷贝：git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia（themes/yilia表示下载目录）

修改config，把theme改为yilia后：hexo clean&&g&&d

butterfly：

git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly，在博客项目根目录运行npm，下载 pug 以及 stylus 的渲染器，否则在项目运行时会报错：npm install hexo-renderer-pug hexo-renderer-stylus –save，修改config，把theme改为butterfly

为了減少升级主题带来的不便，我们还需要做以下操作：
把主题文件夹中的 config.yml复制到 Hexo 根目录下，同重命名为 config.butterfly.yml

Hexo会自动合并主题中的config.yml和 config.butterfly.yml里的配置，如果存在同名配置，会使用config.butterfly.yml的配置，其优先度较高，以后只需在后者中修改。

查看别人的主题：

右击，查看网页源码，
ctrl+ F 搜索关键字 theme，尤其是里面的hexo-theme

6.图片不显示怎么办

修改config文件：post_asset_folder: true，这样在执行hexo new 的时候会生成一个同名文件夹来存放图片，图片命名尽量用英文或数字，且插入到Typora时要用相对路径命名，例如“/用hexo-github搭建博客/1.png”

安装插件：npm install https://github.com/CodeFalling/hexo-asset-image – save
